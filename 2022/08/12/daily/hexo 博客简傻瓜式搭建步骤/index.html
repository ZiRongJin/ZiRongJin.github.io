hexo 博客傻瓜式搭建步骤
kaneko Lv1

Note: 按照以往经验，对于这种涉及多步操作的工作，可能当时记得清楚，但是过一段时间后就会忘

需要提前安装好的工具

具体的步骤可以参考hexo github 搭建博客准备工作

  • nodejs
  • 用nodejs安装hexo
  • git
  • github建立个人主页的仓库
  • notepad++

傻瓜式创建步骤

  1. 新建一个空文件夹，我命名为Blog
  2. 进入该文件夹，单击右键，点git bash
  3. 命令行输入指令：
    • hexo init 由于github是国外网站，第一次可能会因为网络问题出错，多输入几次命令行就行了
    • hexo npm install hexo-deployer-git --save
    • hexo g 生成更新博客文件
    • hexo s 本地查看，方便调试
  4. 以notepad++打开 Blog下的 config.yml 文件
    • 可以将title自定义
    • url改成github homepage网址，比如我的个人博客主页网址是https://zirongjin.github.io/
    • 翻到最下面deploy那里，将下面的代码拷贝过去替换默认的deploy属性。注意，把github.com那个地方的地址改成你自己的homepage网址。最后按Ctrl+S快速保存
  5. 回到git bash界面，输入hexo g -d按回车后，打开个人网站，就能看到一个初步的博客样式啦！接下来是导入博客主题以及自定义

导入theme（以keep theme为例）

  1. 依然是在git bash 里，输入一下指令下载keep theme (其他theme可以在https://hexo.io/themes/ 中挑选)
    • npm install hexo-theme-keep
  2. 以notepad++打开Blog下的 config.yml 文件,翻到最下面，找到theme: 属性，将默认的landscape改成keep，保存文件
  3. 回到git bash界面，输入hexo g -d按回车后，打开个人网站，就能看到博客风格变成了keep样式。(服务器部署需要一段时间，所以可能刚打开的时候依然是landscape theme，可以通过在git bash输入hexo s快速查看)
  4. 关于keep theme样式自定义，可以打开Blog\node_modules\hexo-theme-keep 目录下的 config.yml 文件(用NotePad++打开)，按照keep的配置指南即可进行keep主题内的自定义，更改属性后记得保存

关于博客存放的位置以及更新发表

文章以markdown的格式(.md)存在Blog\source_posts下面，可以用markdown的编辑软件(如obsidian)在该文件夹下创建一个仓库，然后直接用编辑软件新建md博客文章，写好之后再用git bash的指令hexo g -d进行更新发布，以及最开始的hello-world.md不要乱动(可以改里面内容，但是不要把该文件移到别的位置去)
